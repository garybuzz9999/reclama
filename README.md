# 📺 AdScreen — Платформа рекламних моніторів

> Система керування рекламним контентом для планшетів і телевізорів у таксі, кафе, спортзалах, клініках та будь-яких локаціях з captive audience.

---

## 🗂 Структура проекту

```
adscreen/
├── index.html              ← Головна сторінка (редірект)
├── admin/
│   └── index.html          ← Супер-адмін панель (ти)
├── client/
│   └── index.html          ← Кабінет клієнта (рекламодавець)
├── player/
│   └── index.html          ← Плеєр для планшета / телевізора
└── README.md               ← Цей файл
```

4. Натиснути **Додати**
5. Система автоматично генерує ліцензійні коди вигляду:

   ADS-3K9F-WXRT-002
   ADS-3K9F-WXRT-003
   ```
6. Натиснути 🔑 навпроти клієнта → скопіювати коди → передати клієнту

## 🏢 Кабінет клієнта

**Файл:** `client/index.html`

**URL:** `https://YOUR_NAME.github.io/adscreen/client/`

**Демо-вхід для тестування:**
```
Email:    pizza@center.ua
Пароль:   1234
```

### Розділи кабінету

#### 📊 Огляд
- Кількість активних пристроїв
- Кількість опублікованих банерів
- Покази та QR-сканування за сьогодні
- Список ліцензійних кодів (вільні / активовані)

#### 📱 Пристрої
Підключення нового планшета або телевізора:

1. Відкрити кабінет клієнта на ПК або телефоні
2. Перейти у вкладку **Пристрої**
3. Вибрати вільний ліцензійний код зі списку
4. Ввести назву пристрою (наприклад: `Таксі #12` або `Зал А`)
5. Вибрати **зону** — де знаходиться пристрій:
   - 🏙 Центр
   - ⬆️ Північ
   - ⬇️ Південь
   - ⬅️ Захід
   - ➡️ Схід
6. Натиснути **Активувати**

Тепер на планшеті відкрити плеєр і ввести той самий код.

#### 🖼 Банери
Завантаження рекламного банера:

1. Ввести назву банера
2. Вибрати тривалість показу: 15 / 20 / 30 / 45 / 60 секунд
3. Вибрати зони показу (можна кілька або **Всі зони**)
4. Завантажити зображення (JPG / PNG, рекомендований розмір **1920×1080**)
5. Натиснути **Опублікувати банер**

Банер одразу з'явиться на всіх активних пристроях обраних зон.

> 💡 **Порада для рекламодавця:** додайте QR-код прямо на зображення банера — клієнти зможуть сканувати і одразу переходити на сайт / меню / замовлення.

#### 📅 Розклад
- Автоматичний розклад показів протягом дня
- Налаштування паузи між банерами: 3 / 5 / 10 / 15 хвилин
- Вибір контенту між рекламою: мультики / музика / чорний екран

#### 📈 Статистика
- Загальна кількість показів
- QR-сканування (переходи)
- Конверсія (%)
- Графіки по зонах і по кожному банеру

## 📺 Плеєр (планшет / телевізор)

**Файл:** `player/index.html`

**URL:** `https://YOUR_NAME.github.io/adscreen/player/`

### Перше налаштування планшета

1. Встановити **Fully Kiosk Browser** (Android)
   - Google Play: `Fully Kiosk Browser & App Lockdown`
   - Ціна: $14 (одноразово)

2. У налаштуваннях Fully Kiosk:
   ```
   Start URL: https://YOUR_NAME.github.io/adscreen/player/
   Autostart: ON
   Keep Screen On: ON
   Motion Detection: ON (вимикати екран без руху)
   Remote Admin: ON (керування з адмінки)
   ```

3. Відкрити плеєр → ввести ліцензійний код → вибрати зону → **Старт**

### Як виглядає плеєр

```
┌─────────────────────────────────┐
│                                 │
│     [Банер / Мультик / Музика]  │
│                                 │
├─────────────────────────────────┤
│ 🎵 Стиль музики:                │
│ [Поп] [Джаз] [Рок] [Класика]   │
│              [🔇 Вимкнути]      │
└─────────────────────────────────┘
```

Клієнт (пасажир / відвідувач) може:
- Вибрати стиль музики
- Вимкнути звук

### Логіка показу контенту

```
Старт поїздки (водій вибрав зону)
↓
Мультик / фонова музика — 3-5 хв
↓
Банер рекламодавця — 30 сек (тільки для цієї зони!)
↓
Мультик / музика — 3-5 хв
↓
Наступний банер — 30 сек
↓
... і так по колу
```

### Підключення через Chromecast / Miracast

Якщо є звичайний телевізор без Android:

```
Варіант А — Chromecast ($30):
Смартфон → Chrome → "Транслювати вкладку" → ТВ

Варіант Б — Miracast ($15):
Смартфон → налаштування → "Бездротовий дисплей" → ТВ

Варіант В — Amazon Fire Stick ($40):
Відкрити Silk Browser → ввести URL плеєра → кіоск-мод
```

## 🗺 Зони показу

Система ділить місто на 5 зон відносно центру:

```
        ⬆️ ПІВНІЧ
         │
⬅️ ЗАХІД ─┼─ СХІД ➡️
         │
       ⬇️ ПІВДЕНЬ

        🏙 ЦЕНТР (окрема зона)
```

Рекламодавець платить тільки за потрібні зони.

**Таргетинг без GPS:**
```
Водій натискає "Їдемо на Центр"
↓
Планшет показує рекламу кафе / ресторанів / магазинів ЦЕНТРУ
↓
Клієнт сканує QR → замовляє піцу по дорозі додому
↓
Доставка і клієнт приїжджають майже одночасно 🍕
```

## 🔐 Безпека

> Поточна версія використовує `localStorage` для зберігання даних — це MVP для демонстрації. Перед production-запуском обов'язково підключити Firebase Authentication та Firestore.

Для production:
1. Firebase Auth — реєстрація/вхід клієнтів
2. Firestore Rules — клієнт бачить тільки свої дані
3. Cloudflare R2 — публічний бакет тільки для читання файлів
4. HTTPS — GitHub Pages надає автоматично
